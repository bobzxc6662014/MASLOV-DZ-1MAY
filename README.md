# Управление BMX-магазином

Этот проект представляет собой систему для учета товаров, работников и филиалов BMX-магазина с использованием базы данных SQLite. Она позволяет удобно отслеживать ассортимент, персонал и филиалы магазина.

---

## Основные возможности
- Добавление, редактирование и удаление информации о работниках
- Учет наличия и характеристик велосипедов
- Управление филиалами магазина
- Учет выдачи велосипедов в различные филиалы
- Просмотр всей информации и ведение учета

---

## Структура базы данных
База данных содержит следующие таблицы:

### 1. workers (Работники)
- id — уникальный идентификатор сотрудника
- name — ФИО
- post — должность
- department — отдел
- work_experience — стаж работы (в годах)
- salary — зарплата

### 2. bikes (Велосипеды)
- id — уникальный идентификатор
- series — серия модели
- type — тип велосипеда (например, "начальный", "средний", "профессиональный")
- current_type — текущий статус типа
- cost — цена велосипеда
- prodano — количество проданных единиц

### 3. magazin (Филиалы)
- id — уникальный идентификатор филиала
- nomer — номер филиала
- country — страна
- city — город

### 4. borrowings (Выдачи)
- id — уникальный идентификатор
- magazin_id — ссылка на филиал
- bikes_id — ссылка на велосипед
- date_borrowed — дата выдачи
- date_returned — дата возврата (может быть пустой, если велосипед не возвращен)

## Технологии
- **Язык программирования:** Python 3
- **База данных:** SQLite
- **Инструменты:** SQLite, редактор кода (например, VSCode), GitHub для хранения кода

## Использование
1. Скрипт полностью автоматизирован: создает таблицы и наполняет их начальными данными.
2. Для работы с базой рекомендуется написать отдельные скрипты или интерфейсы для добавления, удаления и поиска данных.
3. В будущем можно расширить проект, добавив функции поиска, фильтрации и отчетности.
